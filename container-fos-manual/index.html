Container FOS操作手册

（FortiGate-Docker）

环境介绍​

本次环境部署在Ubuntu 22.04.5 LTS (GNU/Linux 6.8.0-51-generic x86_64)

Docker版本为：

  • Version: 27.5.1
  • API version: 1.47
  • Go version: go1.22.11
  • Git commit: 9f9e405
  • Built: Wed Jan 22 13:41:31 2025
  • OS/Arch: linux/amd64
  • Context: default

上传镜像文件到服务器​

通过使用scp命令，将本地镜像发送到服务器中，服务器需要开启ssh才能使用scp，命令示例：

scp -r FOS_X64_DOCKER-v7-build0255-FORTINET.tar.gz admin@192.168.0.1:/home/username/
  1. scp是命令，
  2. -r是需要上传的文件名称
  3. admin@192.168.0.1是要上传的服务器用户名和 IP 地址
  4. /home/username/是要拷入的服务器文件夹路径

加载镜像​

sudo docker load -i FOS_ARM64_DOCKER-v7-build0230-FORTINET.tar.gz

查看是否正常加载使用docker image ls 查看

创建容器网卡​

docker network create --driver=bridge --subnet=10.0.20.0/24 lan0

命令解释：

  • --driver 网卡类型
  • --subnet=10.0.20.0/24 表示网卡的网段也表示FortiGate使用的网段
  • lan0 表示docker容器中网络的名称

启动容器并连接网卡​

docker run -d \
-v /etc/localtime:/etc/localtime:ro \
-v /text:/data \
--cpus 2 \
--memory 1024m \
--cap-add=NET_RAW \
--cap-add=NET_ADMIN \
--cap-add=SYS_ADMIN \
--name fos \
-p 8080:80 \
-p 500:500/udp \
-p 4500:4500/udp \
--net=lan0 \
Fos

命令解释：

  • --net= 这里写的是我们刚才创建的网络名称
  • -v /etc/localtime:/etc/localtime:ro：将宿主机的本地时间文件挂载到容器，确保容器使用宿主机的时间
  • -v /text:/data：将宿主机的 /text 目录挂载到容器的 /data 目录
  • --cpus 2：限制容器使用最多 2 个 CPU 核心
  • --memory 1024m：限制容器最多使用 1GB 的内存
  • --cap-add=NET_RAW 和其他 --cap-add 参数：为容器添加特权权限，使容器能够进行网络和系统管理
  • -p 8080:80：将宿主机的 8080 端口映射到容器的 80 端口（通常是 HTTP 服务）
  • --net=lan0：指定容器连接的网络为 lan0，确保容器使用正确的网络接口
  • fos：指定要运行的 Docker 镜像名

导入许可​

运行完上一步的命令之后会自动启动FortiGate，用户名是admin密码为空直接回车，启动后使用

exe import-vmlicense "license内容全选复制粘贴到这"

导入license文件

如果卡在 System is ready界面不要慌，重新打开一个页面输入

sudo docker exec -it fos /bin/cli

这条命令是进入FortiGate的命令行界面

  • -it fos 中的fos表示容器的名称

然后再使用

exe import-vmlicense "license内容全选复制粘贴到这"

导入license文件

粘贴进去之后会提示证书导入成功，稍等片刻会提示证书更新，更新完使用

docker exec -it fos /bin/cli

重新进入FortiGate 命令行

进去之后就到了熟悉的命令行界面了

其他​

进入FortiGate 底层 命令：

sysctl sh

Exit返回到fortigate cli

常用Docker命令：

  • Docker ps -a / 查看容器
  • Docker network ls / 查看网络
  • Docker rm stop 容器名称 / 关闭容器
  • Docker rm 容器名称 / 删除容器
  • Docker network rm 网络名称 / 删除网络

需要删除容器，需要先删除关联容器的网络，然后停止容器，再删除容器。

Exit 命令可以从底层切到cli中，也可以从cli切到物理机命令行。

License文件是需要从support网站上注册后下载打开。
